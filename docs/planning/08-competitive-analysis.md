# 경쟁 분석

## 직접 경쟁: 자기소개 페이지들 (MVP 챌린지 멤버)

| 사이트 | 강점 | 약점 | 우리와의 차이 |
|--------|------|------|-------------|
| **wlsvhr** (상상력설계사) | 콘텐츠 깊이, AI 파트너 내러티브, 앰비언트 사운드 | 보기만 하는 페이지, 소통 수단 부재 | 우리는 키워드/문장 단위로 반응 가능 |
| **selfservice** (19년차 개발자) | 글래스모피즘, 숫자 스탯, 시각 완성도 | 전형적 포트폴리오 구조, 서사보다 스펙 | 우리는 서사/관심사 중심 |
| **KAIROS** (에디토리얼) | 매거진 레이아웃, 타이포 세련 | 단방향 소개, 탐색 구조 없음 | 우리는 폴더트리로 깊이 탐색 |
| **Life OS** (OS 메타포) | 세계관 판매, 카피라이팅 | 자기소개가 아닌 제품 랜딩, 소통 없음 | 우리도 세계관 + 하지만 대화까지 |

## 간접 경쟁: 기존 플랫폼

| 플랫폼 | 우리와의 차이 |
|--------|-------------|
| **Obsidian Publish** | 개인 지식 베이스 퍼블리싱만 가능, 소셜 레이어 없음 |
| **Digital Garden / Quartz** | 정적 사이트 생성, 댓글/반응 시스템 없음 |
| **Are.na** | 비주얼 북마크/연결, 그러나 서사 중심이 아님 |
| **LinkedIn** | 이력서/스펙 중심, 관심사/서사 구조 없음 |
| **Medium / 블로그** | 시계열 포스팅, 구조화된 서사가 아님 |
| **X (Twitter)** | 순간 스냅샷, "나 자체"를 설명하지 못함 |

## 차별화 요약

```
기존 자기소개 페이지:  "보세요" → 끝
연결의 숲:           "보세요" → "이거 관심 있어요" → "이 주제로 대화해요"

기존 SNS:            시계열 포스팅 → 좋아요/댓글
연결의 숲:           구조화된 서사 퍼블리싱 → 키워드 반응/문장 댓글/주제 메시지

기존 지식 베이스:     개인 메모 퍼블리싱
연결의 숲:           서사 퍼블리싱 + 소셜 레이어
```
